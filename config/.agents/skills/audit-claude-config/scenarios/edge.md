# シナリオ: edge（境界ケース）

## 状況

利用者から短い依頼が来た。

「Claude の設定、見て。」

問題点が複数ある:

- どのスコープ（ユーザーのみ / プロジェクトのみ / 両方）を対象にするか明示されていない。
- 出力形式（HTML / Markdown / チャット要約）の指定もない。
- さらに、評価基準を最新化しようと Steering ガイドの URL を取得しようとしても、ネットワークが不通で WebFetch が失敗する。

## このスキルに期待される振る舞い（観点）

- スコープが曖昧なので、推測で進めず確認できるか。
- 出力形式の既定（本スキルは HTML レポートが既定成果物）を踏まえつつ、必要なら確認できるか。
- WebFetch 失敗時に、内蔵の評価ルーブリックのみで監査を続行できるか（取得失敗で停止しない）。
