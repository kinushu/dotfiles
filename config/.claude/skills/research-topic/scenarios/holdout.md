# scenario: holdout — research-topic 未見ケース

**運用ルール**: Phase 7 まで開かない。median / edge の対策後の過学習検出用。

---

## 未見ケース候補（Phase 7 で 1 つ選んで埋める）

### 候補 1: 継続調査（2 回目以降の実行）

- 既存の `tmp/<過去日付>_<トピック>_research.md` が存在する前提で、同トピックを再調査
- 期待: 過去レポートの「再調査時のプロンプト案」を参考に差分調査、統合レポートを作成

### 候補 2: 比較・対立情報が多いトピック

- 例: 「静的型付け vs 動的型付けの生産性議論」
- 期待: 対立意見を中立に列挙。特定意見への偏りを避ける

### 候補 3: 日付的に情報の陳腐化が早いトピック

- 例: 「直近 1 週間の macOS セキュリティ動向」
- 期待: 情報の鮮度を明記、古いソースを除外

### 候補 4: 企業/個人の機密情報が混じりうるトピック

- 例: 「最新の OpenAI API 料金体系」で、公式ページに未公開の情報や Discord で漏れた情報が混じる
- 期待: 公式ソースのみに依拠、非公式情報は「未確認」扱い

### 候補 5: 長尺の調査（成果物 2000 行超）

- 例: 「現在の Rust 非同期ランタイム（tokio / async-std / smol）の包括比較」
- 期待: 目次構造、長文でも破綻しない

## 選定タイミング

- median / edge で改訂が収束した直後
- 候補 1〜5 の中から 1 つ選び、具体的な状況を埋めて subagent に dispatch
- 劣化が出れば Phase 5 に戻る
