# モデル分業委託スキル（delegate-by-model）

## 概要

ADR や計画書（.md）を伴う規模の大きいタスクでは、工程をモデルで分業する。

- **解析・設計・計画書作成**: 現モデル（このセッションのモデル）が担う。
- **実装**: 確定した計画書を入力として、サブエージェントへ委託する。現モデルは原則として直接実装しない。

本スキルは「実装を委託する」フェーズの手順を定める。計画書が未確定の状態では起動しないこと。

## 前提条件

- 実装対象の計画書（またはADR）が確定し、利用者の承認を得ていること。
- 計画書の内容が曖昧・未確定の場合は本スキルを起動しない。

## 手順

### 0. 起動可否ゲート（前提条件チェック）

手順に入る前に、前提条件（計画書 / ADR が確定・承認済み）を満たすか判定する。

- 計画書 / ADR が未確定・曖昧・未承認なら、**本スキルを起動せず**、計画フェーズ（解析・設計・計画書作成。現モデルが担う）へ差し戻す。以降の手順は実行しない。
- 前提を満たす場合のみ手順 1 へ進む。

### 1. 委託モードを確認する

**委託前**に、AskUserQuestion で **委託モード** を確定する（本スキルは委託フェーズ専用。計画着手前のモード確認は上流ルール側の責務であり、ここでは委託前確認のみを行う）。ただし **同一セッション内で「よしなに一任」が既に事前承認済みなら、本手順の質問（1 段目）も省略**してよい（モードは確定済みのため、そのまま手順 2 の自動選定へ進む）。

- **明示選択モード**（想定モデルがある / 自分で選びたい）: 手順 2 で委託先カテゴリを AskUserQuestion で選ばせる。
- **よしなに一任モード**（想定なし / 任せる）: 対話を省き、現モデルが手順 2 の基準で自動選定する。

> **AskUserQuestion の段数**: 質問は最大 2 段。まず本手順でモードを確定し（一任が事前承認済みなら 1 段目を省略）、明示選択モードの場合のみ手順 2 でカテゴリを尋ねる。よしなに一任モードなら 2 段目は省略する。
>
> **「事前承認済み」の判定基準**: 同一セッション内で利用者が「よしなに一任」を明示的に表明した記録がある状態を指す。痕跡が無ければ未承認（= 明示選択が必要）とみなす。
>
> **非対話実行時の規範**: AskUserQuestion を提示・応答できない実行形態（非対話実行）では、「よしなに一任」が事前承認済みなら自動選定で続行し、「明示選択モード」またはモード未確定なら委託に進まず停止して明示選択を求める。既定モデルでの黙委託はしない。

### 2. 委託先カテゴリを決定する

上流で確定するのは **委託先カテゴリ**（`opus` / `sonnet` 等、または `codex`）まで。

- `codex` を選んだ場合の具体モデル（例: `gpt-5-codex`）・推論レベルは `implement-by-codex` 側で確定する。
- 候補は **実行時に利用可能なモデルから動的に列挙** する（固定のモデル名を直書きしない）。
- 列挙元は Claude Code が把握できる alias / 設定上の候補と、利用者が明示した外部 CLI 候補。
- **完全列挙は保証できないため、代表候補に加えて「その他（自由入力）」を必ず併設** し、列挙漏れに備える。
- 役割の目安: 「高知能機（例: Opus）/ 標準機（例: Sonnet）/ 外部CLI（例: Codex）」。具体的なモデル名は例示であり、各社の世代交代に追従して都度入れ替える。
- **よしなに一任モードの自動選定の目安**（タスク特性 → カテゴリ）:
  - 込み入ったアルゴリズム / 設計判断 / 広範な影響・高い破壊度を伴う → 高知能機（例: Opus）
  - 定型的な多ファイル編集 / ボイラープレート / 明確な手順の反復 → 標準機（例: Sonnet）
  - 利用者が外部 CLI を指定済み、または別 CLI の特性が要る → 外部CLI（例: Codex、`implement-by-codex` 経由）
  - 判断材料の目安: 変更規模（行数・ファイル数）・新規ファイルの有無・対象言語・設計自由度・破壊度。迷う場合は上位カテゴリ（高知能機）に倒す。
- **既定は計画単位で同一カテゴリ**。実装が混在するタスクでは、必要に応じサブタスク単位で再選定してよい。

### 3. 委託先に応じて起動方法を選ぶ

| 委託先カテゴリ | 起動方法 |
|---|---|
| `codex` | `implement-by-codex` スキル経由で起動する |
| `opus` / `sonnet` 等（Claude 系） | Agent ツールの `model` 上書き（`opus` / `sonnet` 等）で起動する |

> **環境変数の優先注意**: `CLAUDE_CODE_SUBAGENT_MODEL` 等の環境変数が設定されていると、per-invocation の `model` 上書きより優先される場合がある。委託前に環境変数の有無を確認すること。

### 4. 委託プロンプトを構成する

- 確定済み計画書の本文を入力として含める。
- 利用可能な場合は `implementer` subagent 定義（`~/.claude/agents/implementer.md`）を参照させる。
- 計画書に記載のないファイルは変更しないよう明示する。
- 実装後に変更ファイル一覧と概要を報告させる。

### 5. 委託後の検証（必須）

実装完了後、必ず以下を実施する。問題が見つかれば中断してユーザーへ報告する。

- `git diff` で変更内容を確認する。
- `make test` または `make deploy-dry-run` で環境を検証する。
- 検証に問題があれば即座に中断し、エラー内容を報告する。自動修正は行わない。

## 適用範囲

- 本スキルは **Claude Code 限定**。Codex CLI / Gemini CLI ではネイティブサブエージェントのモデル上書き機構が無いため適用しない。
- **小規模タスク**（計画書を作らない軽微な修正等）には本スキルを適用しない。現モデルが直接実装してよい。
