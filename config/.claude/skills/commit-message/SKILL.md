# コミットメッセージ生成

## 手順

1. `git diff --staged` で変更内容を確認
2. 以下の形式で日本語メッセージを生成:
   - 1行目: 変更の要約（50文字以内）
   - 空行
   - 詳細説明（なぜこの変更が必要か）

## フォーマット例

```
機能追加: ユーザー認証にOAuth対応を追加

GoogleとGitHubアカウントでのログインを可能にした。
既存のメール認証との併用も可能。
```

## 注意点

- 「何を」ではなく「なぜ」を重視
- 動詞は現在形（追加、修正、削除など）
- 技術的な詳細より意図を伝える
