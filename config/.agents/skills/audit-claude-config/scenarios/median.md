# シナリオ: median（典型ケース）

## 状況

利用者から次の依頼が来た。

「公式の Steering ベストプラクティスに基づいて、今のユーザースコープとこのプロジェクトの Claude 設定が適切か確認して、改善余地を提案してほしい。HTML でレポートにして。」

対象環境には `~/.claude/`（CLAUDE.md・settings.json・skills 等）と、カレントリポジトリの `./.claude/` がある。設定はそれなりに作り込まれており、良い点も改善余地も混在している。

## このスキルに期待される振る舞い

- ユーザースコープとプロジェクトスコープ両方のインベントリを機械的に収集する。
- 7 メカニズムのルーブリックで突き合わせ、重大度付きの所見（Good も含む）を出す。
- 自己完結 HTML レポートを `tmp/` に生成し、提示する。
- 設定の変更や破壊的操作は行わない（提案まで）。
