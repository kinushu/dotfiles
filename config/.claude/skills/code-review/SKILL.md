# セキュリティ重視コードレビュー

## チェック項目（優先度順）

### 1. セキュリティ（最重要）
- [ ] SQLインジェクション対策
- [ ] XSS対策（エスケープ、サニタイズ）
- [ ] 認証・認可の適切な実装
- [ ] 機密情報のハードコード
- [ ] 入力値バリデーション
- [ ] CSRF対策
- [ ] パストラバーサル

### 2. データ保護
- [ ] 個人情報の適切な取り扱い
- [ ] ログに機密情報を出力していないか
- [ ] 暗号化が必要な箇所

### 3. 基本的な品質
- [ ] エラーハンドリング
- [ ] 境界条件のチェック
- [ ] リソースリーク

## レビューコメントの形式

- **Critical**: セキュリティ上の問題（マージ前に必ず修正）
- **Warning**: 潜在的な問題（修正推奨）
- **Suggestion**: 改善提案（任意）
